## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.10 · `prd/ui/navigation-layout-spec.md`

**目标**：从左侧边栏导航迁移为 Rail + Sidebar + Main 三段式布局，为后续 Topic 页面和多页面产品建立统一的布局基础。

### 现状分析

当前布局为两栏结构（`LocalPage`，`src/layout/Local/index.tsx`）：

```
App.tsx (Theme wrapper)
└── LocalPage (不在路由树中，直接渲染)
    ├── 左侧栏 (w-[--app-feedlist-width: 230px])
    │   ├── 工具栏 (48px): SpaceSwitcher + AddFolder + Refresh
    │   ├── 快捷入口: Search / New Subscribe / Settings (NavLink + Dialog)
    │   └── ChannelList (DndProvider 拖拽排序)
    └── <Outlet /> (路由内容)
```

关键约束：
- `LocalPage` **不在路由树中**（`index.tsx` 中 LOCAL 嵌套路由已注释掉），而是通过 `App.tsx` 直接渲染
- Settings 是 Radix `<Dialog>` 弹窗（`settingDialogStatus` store 控制），不是路由页面
- FreshRSS 有独立两栏布局，不走 `LocalPage`
- `SearchPage` 和 `ArticleView` 中有 `100vh` 硬编码高度
- 项目**没有**使用 Zustand `persist` 中间件，localStorage 全部手动管理

**功能范围**：
- 实现 Rail（48px 宽度）包含 6 个一级导航图标
- 实现 Sidebar（240px 宽度）包含品牌区、快捷操作、ChannelList、占位区域
- 实现 Main（flex-1）作为主内容区
- Sidebar 支持收起/展开，收起后只保留 Rail + Main
- 收起/展开状态记忆到 localStorage
- 各现有页面适配新布局，修复 `100vh` 硬编码
- 新增 Topics 占位路由（空状态页面）
- 页面默认路由保持 `/local/today`

**明确不做**：
- ❌ Sidebar 中 Today Focus / Tracked Topics 的实际数据绑定（v2.11+）
- ❌ 页面切换动画（v2.x 阶段不做）
- ❌ 暗色模式调整（已有基础设施，新组件使用 CSS 变量/Tailwind token 即可）
- ❌ 响应式断点（< 1024px 自动收起可延后）
- ❌ FreshRSS 页面适配（保留独立布局，不受影响）
- ❌ 任何 Rust 后端变更、新 DB 表、新依赖

**验收标准**：
1. 应用启动后显示 Rail + Sidebar + Main 三段式布局
2. Rail 中 6 个导航图标（Today / Topics / Feeds / Search / Starred / Settings）正确显示
3. 点击 Rail 图标切换对应页面（Today→`/local/today`、Feeds→`/local/all`、Search→`/search`、Starred→`/local/starred`、Topics→`/local/topics`、Settings→触发 Dialog）
4. 当前页面在 Rail 中高亮
5. Sidebar 可收起/展开，收起/展开状态在重启后保持
6. Today / Feeds / Settings 页面在 Main 区域内正确渲染，无布局破损
7. Topics / Search 页面在无数据时显示合理的空状态（不崩溃）
8. ChannelList（订阅源树 + DnD 拖拽）在 Sidebar 中正常工作
9. SearchPage 和 ArticleView 中 `100vh` 硬编码已修复

**成功指标**：
- 布局切换后各页面渲染正常，零布局 bug
- Sidebar 收起/展开操作 < 200ms
- 用户可在 5 秒内理解新的导航结构

## §2 技术方案

### 2.1 数据库变更

无。本版本不涉及后端变更。

### 2.2 前端架构变更

#### 2.2.1 布局组件层级

```
App.tsx (Theme wrapper)
└── AppLayout (新组件，替代 LocalPage)
    ├── Rail (新组件, 48px)
    │   ├── 品牌图标 (Lettura Logo)
    │   └── RailItem × 6 (Today/Topics/Feeds/Search/Starred/Settings)
    ├── Sidebar (新组件, 240px)
    │   ├── SidebarHeader — 品牌文字 + 收起按钮
    │   ├── SidebarActions — Search / Add Subscribe / Add Folder / Refresh 快捷操作
    │   ├── ChannelList (迁移自 LocalPage，保留 DndProvider)
    │   └── SidebarFooter — 占位区域 (Today Focus + Tracked Topics 占位文本)
    └── Main (flex-1, 新组件)
        └── <Outlet /> (react-router)
```

#### 2.2.2 新增文件

| 文件路径 | 职责 |
|---------|------|
| `src/components/layout/AppLayout.tsx` | 三段式布局容器，替代 `LocalPage` 的布局职责 |
| `src/components/layout/Rail.tsx` | 左侧 Rail 导航（48px 固定宽度） |
| `src/components/layout/Sidebar.tsx` | 中间 Sidebar（240px，可收起） |
| `src/components/layout/Main.tsx` | 右侧主内容区（flex-1） |
| `src/components/layout/RailItem.tsx` | 单个 Rail 导航项（图标 + tooltip + active 状态） |
| `src/pages/TopicsPage.tsx` | Topics 占位页面（空状态） |

#### 2.2.3 路由变更

路由结构保持扁平（不引入嵌套 `AppLayout` 路由），因为 `LocalPage` 不在路由树中，而是由 `App.tsx` 直接渲染。改造方式为替换 `App.tsx` 中的 `<LocalPage />` 为 `<AppLayout />`：

```typescript
// 当前（App.tsx 简化）
<Theme>
  <LocalPage />       {/* 直接渲染，包含 Outlet */}
  <DialogAboutApp />
</Theme>

// 新版（App.tsx 简化）
<Theme>
  <AppLayout>         {/* 替代 LocalPage */}
    <Outlet />
  </AppLayout>
  <DialogAboutApp />
</Theme>
```

路由定义（`index.tsx`）新增 Topics 占位路由：

```typescript
// 新增
{
  path: "/local/topics",
  element: <TopicsPage />,   // 占位空状态页面
},
```

#### 2.2.4 Rail 导航项定义

| 图标 (lucide-react) | 名称 | 路由 / 行为 | 说明 |
|---------------------|------|------------|------|
| `Sun` | Today | `navigate('/local/today')` | 默认入口，Active 最突出 |
| `Layers` | Topics | `navigate('/local/topics')` | v2.10 占位空状态页 |
| `Rss` | Feeds | `navigate('/local/all')` | 所有订阅源文章 |
| `Search` | Search | `navigate('/search')` | 搜索页 |
| `Star` | Starred | `navigate('/local/starred')` | 收藏文章 |
| `Settings` | Settings | `store.updateSettingDialogStatus(true)` | 触发 Settings Dialog（非路由） |

**注意**：Settings 不走路由导航，而是触发已有的 Radix Dialog。RailItem 需要支持两种模式：路由模式和回调模式。

#### 2.2.5 状态管理

Sidebar 展开/收起状态添加到 `UserConfigSlice`（跟随项目现有手动 localStorage 模式）：

```typescript
// createUserConfigSlice.ts 新增
interface UserConfigSlice {
  // ... 现有字段
  sidebarCollapsed: boolean;
  toggleSidebar: () => void;
}

// 初始化：从 localStorage 读取
sidebarCollapsed: localStorage.getItem("sidebar_collapsed") === "true",

// 切换：更新 state + 写入 localStorage
toggleSidebar: () => set((state) => {
  const next = !state.sidebarCollapsed;
  localStorage.setItem("sidebar_collapsed", String(next));
  return { sidebarCollapsed: next };
}),
```

当前页面高亮状态从 react-router 的 `useLocation` 推导。

#### 2.2.6 需要修改的现有文件

| 文件 | 变更 |
|------|------|
| `src/App.tsx` | `<LocalPage />` → `<AppLayout><Outlet /></AppLayout>` |
| `src/index.tsx` | 新增 `/local/topics` 路由，import `TopicsPage` |
| `src/stores/createUserConfigSlice.ts` | 新增 `sidebarCollapsed` + `toggleSidebar` |
| `src/layout/Search/index.tsx` | 修复 `100vh` → `h-full` |
| `src/layout/Article/View.tsx` | 修复 `100vh` → flex 相对高度 |
| `src/styles/reading-style.css` | `--app-feedlist-width` 保留但不再使用（或改为 `--app-sidebar-width: 240px`） |

#### 2.2.7 需要移除的现有组件/代码

| 组件 / 代码 | 原因 |
|------------|------|
| `src/layout/Local/index.tsx` | 被 `AppLayout` 替代 |
| `src/components/SpaceSwitcher/index.tsx` | Rail 顶部品牌图标替代其功能（仅有一个空间 "Lettura"，无需切换器） |
| `src/styles/reading-style.css` 中 `--app-feedlist-width` | 被 Rail(48px) + Sidebar(240px) 替代 |

**注意**：`MainPanel` 组件保留不动，各页面继续使用它作为内容包裹器。

## §3 交互规格

### 3.1 Rail 交互

| 操作 | 行为 |
|------|------|
| 点击导航图标 | 路由模式：切换到对应路由；回调模式：执行回调（如 Settings Dialog） |
| Hover 导航图标 | 显示 tooltip（页面名称，使用 Radix `Tooltip`） |
| 当前页面 | 强调色背景（`bg-accent-9`） + 白色图标 |
| 非当前页面 | 透明背景 + `#9CA3AF` 灰色图标 |
| Hover 非当前 | `#F5F5F5` 浅灰背景 |

### 3.2 Sidebar 交互

| 操作 | 行为 |
|------|------|
| 点击收起按钮 | Sidebar 收起（200ms ease-out 过渡），只保留 Rail + Main |
| 点击展开按钮（在 Rail 底部或品牌区） | Sidebar 展开 |
| 收起/展开 | 状态记忆到 localStorage |
| ChannelList 拖拽 | 保留完整 DnD 功能 |

### 3.3 Sidebar 内容（本版本）

- **Header**：Lettura 品牌文字 + Sidebar 收起按钮
- **快捷操作区**：Search / New Subscribe / Add Folder / Refresh 按钮（从现有 LocalPage 工具栏迁移）
- **ChannelList**：订阅源树形列表 + DnD 拖拽排序（完整保留）
- **Footer 占位**：
  - Today Focus：显示"Today Focus · 查看最新判断"（静态文本，点击跳转 Today）
  - Tracked Topics：显示"尚无跟踪主题"（v2.11+ 实现）

### 3.4 Settings 交互

Settings 仍通过 Radix Dialog 弹窗实现，不受布局迁移影响。Rail 中 Settings 图标点击触发 `store.updateSettingDialogStatus(true)`。

## §4 视觉规范

遵循 `prd/ui/visual-system-spec.md`：

- Rail 背景：`#FAFAFA`（`bg-gray-1`）
- Sidebar 背景：`#FFFFFF`（`bg-panel`），右侧 `#E5E5E5` 细边框（`border-r border-gray-5`）
- 导航图标 Active：`#2563EB` 背景（`bg-accent-9`）+ 白色图标
- 导航图标 Inactive：透明背景 + `#9CA3AF` 图标（`text-gray-9`）
- 导航图标 Hover：`#F5F5F5` 背景（`bg-gray-4`）
- Sidebar 收起/展开过渡：`transition-all duration-200 ease-out`

**使用 Tailwind token**：所有颜色使用已有的 Radix/Tailwind token（`bg-gray-*`、`text-gray-*`、`bg-accent-*`），不硬编码十六进制值，以兼容未来暗色模式。
