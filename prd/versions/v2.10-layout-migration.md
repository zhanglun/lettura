## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.10 · `prd/ui/navigation-layout-spec.md`

**目标**：从顶部 tab 导航迁移为 Rail + Sidebar + Main 三段式布局，为后续 Topic 页面和多页面产品建立统一的布局基础。

**功能范围**：
- 实现 Rail（48px 宽度）替代现有顶部 tab 导航
- 实现 Sidebar（240px 宽度）包含品牌区、Today Focus、Tracked Topics 三个区域
- 实现 Main（flex-1）作为主内容区
- Sidebar 支持收起/展开，收起后只保留 Rail + Main
- 收起/展开状态记忆到 localStorage
- 各现有页面（Today、Feeds、Settings）适配新布局
- 页面默认路由保持 `/today`

**明确不做**：
- ❌ Sidebar 中 Today Focus / Tracked Topics 的实际数据绑定（v2.11 Topic 引入后实现）
- ❌ 页面切换动画（v2.x 阶段不做）
- ❌ 暗色模式
- ❌ 响应式断点（< 1024px 处理可延后）
- ❌ 任何 Rust 后端变更、新 DB 表、新依赖

**验收标准**：
1. 应用启动后显示 Rail + Sidebar + Main 三段式布局
2. Rail 中 6 个导航图标（Today / Topics / Feeds / Search / Starred / Settings）正确显示
3. 点击 Rail 图标切换对应页面
4. 当前页面在 Rail 中高亮
5. Sidebar 可收起/展开，收起/展开状态在重启后保持
6. Today / Feeds / Settings 页面在 Main 区域内正确渲染，无布局破损
7. Topics / Search 页面在无数据时显示合理的空状态（不崩溃）

**成功指标**：
- 布局切换后各页面渲染正常，零布局 bug
- Sidebar 收起/展开操作 < 200ms
- 用户可在 5 秒内理解新的导航结构

## §2 技术方案

### 2.1 数据库变更

无。本版本不涉及后端变更。

### 2.2 前端架构变更

#### 2.2.1 布局组件层级

```
App.tsx
└── AppLayout (新组件)
    ├── Rail (新组件)
    │   └── RailItem × 6
    ├── Sidebar (新组件)
    │   ├── SidebarBrand
    │   ├── SidebarTodayFocus (占位，显示静态文本)
    │   └── SidebarTrackedTopics (占位，显示"尚无跟踪主题")
    └── Main (新组件)
        └── <Outlet /> (react-router)
```

#### 2.2.2 新增文件

| 文件路径 | 职责 |
|---------|------|
| `src/components/layout/AppLayout.tsx` | 三段式布局容器 |
| `src/components/layout/Rail.tsx` | 左侧 Rail 导航 |
| `src/components/layout/Sidebar.tsx` | 中间 Sidebar |
| `src/components/layout/Main.tsx` | 右侧主内容区 |

#### 2.2.3 路由变更

现有路由结构保持不变，仅包裹一层 `AppLayout`：

```typescript
// 当前（简化）
<Route path="/" element={<App />}>
  <Route path="/local/today" element={<Today />} />
  <Route path="/local/feeds" element={<FeedList />} />
  ...
</Route>

// 新版
<Route path="/" element={<App />}>
  <Route element={<AppLayout />}>
    <Route path="/local/today" element={<Today />} />
    <Route path="/local/feeds" element={<FeedList />} />
    ...
  </Route>
</Route>
```

#### 2.2.4 状态管理

- Sidebar 展开/收起状态存入 Zustand store（userConfig slice）+ localStorage
- 当前页面高亮状态从 react-router 的 `useLocation` 推导

#### 2.2.5 需要移除的现有组件

| 组件 | 原因 |
|------|------|
| 顶部 tab 导航相关组件 | 被 Rail 替代 |
| SpaceSwitcher | 已在 UI 重设计中移除 |

## §3 交互规格

### 3.1 Rail 交互

| 操作 | 行为 |
|------|------|
| 点击导航图标 | 切换路由到对应页面 |
| Hover 导航图标 | 显示 tooltip（页面名称） |
| 当前页面 | 强调色背景 + 白色图标 |

### 3.2 Sidebar 交互

| 操作 | 行为 |
|------|------|
| 点击收起按钮 | Sidebar 收起，只保留 Rail + Main |
| 点击展开按钮 | Sidebar 展开 |
| 收起/展开 | 状态记忆到 localStorage |

### 3.3 Sidebar 内容（占位）

本版本 Sidebar 内容为静态占位：

- **品牌区**：Lettura Logo
- **Today Focus**：显示"Today Focus · 查看最新判断"（静态文本，点击跳转 Today）
- **Tracked Topics**：显示"尚无跟踪主题"（Topics 功能在 v2.11+ 实现）

## §6 风险与依赖

| 风险 | 缓解措施 |
|------|---------|
| 现有页面布局依赖顶部导航的结构 | 逐一验证每个页面的布局适配 |
| 窗口缩放时三栏布局可能错位 | v2.10 先不做响应式，最小宽度 1024px |
| Sidebar 内容为空可能显得浪费空间 | 收起为默认状态，用户可展开 |

**前置依赖**：v2.9（深读链路优化）完成，Today 交互链路稳定。

**后续解锁**：v2.11（Topic 最小引入）需要 Sidebar 中的 Tracked Topics 区域。
