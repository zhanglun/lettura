## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.9

**目标**：让用户从 Signal 更顺畅地进入深读——优化点击展开、来源列表交互和阅读器跳转的完整链路，降低误点和重复跳转。同时增强 Search 支持 Signals + Articles 两类结果并列。

**功能范围**：
- 点击 SignalCard 标题 → 展开来源列表（不直接打开文章）
- 来源列表优化：首条为最相关文章，每条来源有清晰的"阅读原文"按钮
- 展开 SignalCard 底部新增"继续阅读"引导区
- 点击来源文章 → 复用现有 ArticleReader（不新开标签页/窗口）
- 从 ArticleReader 返回 → 恢复到同一 SignalCard 的滚动位置（scroll restoration）
- **Search 增强**：搜索结果支持 Signals + Articles 两类并列展示，Signals 优先级高于 Articles（详见 `prd/ui/search-page-spec.md`）

**明确不做**：
- ❌ 新增阅读器组件（复用现有 ArticleReader）
- ❌ 文章内 AI 摘要 / 翻译（后续版本）
- ❌ 离线阅读功能
- ❌ 阅读进度追踪 / 阅读时间统计
- ❌ Search Topics 结果（需要 Topic 数据模型，v2.12 实现）

**验收标准**：
- 从 SignalCard 到打开原文 < 2 次点击
- 从 ArticleReader 返回后，页面滚动位置与离开时一致（偏差 < 50px）
- 不出现重复跳转（同一篇文章不会打开两次）
- 来源列表展开/折叠动画流畅（300ms, ease-in-out）

**成功指标**：
- 深读转化率（Signal → 打开原文）提升 > 20%（相对 v2.5 基线）
- 返回后滚动位置恢复准确率 > 95%
- 无误点（点击标题展开 ≠ 打开文章）投诉 = 0

**ArticleReader 单例跳转**：

```typescript
// 打开文章时，复用现有 ArticleReader 而非新开窗口
const openArticle = (articleId: number, link: string) => {
  // 记录当前滚动位置
  saveScrollPosition();
  // 使用现有的文章阅读路由
  navigate(`/local/article/${articleId}`);
};
```

**Scroll Restoration 机制**：

```typescript
// stores/todaySlice.ts 扩展
interface TodayState {
  // ...existing
  scrollPositionMap: Record<number, number>;  // signalId → scrollY
  expandedSignalId: number | null;            // 当前展开的 SignalCard ID
}

// 保存滚动位置
const saveScrollPosition = () => {
  const scrollY = window.scrollY;
  const expandedId = store.getState().today.expandedSignalId;
  if (expandedId) {
    store.getState().today.setScrollPosition(expandedId, scrollY);
  }
};

// 恢复滚动位置
const restoreScrollPosition = () => {
  const expandedId = store.getState().today.expandedSignalId;
  if (expandedId) {
    const savedY = store.getState().today.scrollPositionMap[expandedId] ?? 0;
    requestAnimationFrame(() => {
      window.scrollTo(0, savedY);
    });
  }
};
```

**防重复跳转**：

```typescript
// 跟踪已打开的文章，防止重复点击
const openedArticles = useRef<Set<number>>(new Set());

const handleOpenArticle = (articleId: number, link: string) => {
  if (openedArticles.current.has(articleId)) return; // 防重复
  openedArticles.current.add(articleId);
  openArticle(articleId, link);
};
```
