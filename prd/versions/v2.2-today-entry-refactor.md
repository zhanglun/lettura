## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.2

**目标**：把 Today 从"今日文章过滤页"升级为产品默认入口。

**功能范围**：
- 调整首页与导航优先级：Today 成为侧边栏第一项（在 Feeds 之上），应用启动后默认路由指向 `/local/today`
- 明确 Today 与 Feed List 的关系：Today 是产品主入口（"先看判断"），Feed List 保留为"按源浏览"的二级入口
- 重写空状态文案（无文章、Pipeline 未运行、无 API Key 分别处理）
- 重写首次进入体验叙事（Onboarding 完成后第一眼看到 Today 的引导）
- 产品叙事文案 i18n 更新

**明确不做**：
- ❌ AI Pipeline / Signal 卡片（2.3）
- ❌ Today Overview 一句话概览（2.8）
- ❌ Why It Matters（2.4）
- ❌ 来源透明 / 展开来源文章（2.5）
- ❌ 任何 Rust 后端变更、新 DB 表、新依赖

**验收标准**：
1. 应用启动后默认进入 Today 页面（而非 Feed List）
2. Today 页面在无 AI 数据时展示优雅的空状态引导
3. 侧边栏 Today 排在 Feeds 之上
4. 所有新增文案支持中英双语

**成功指标**：
- 侧边栏导航点击 Today 的比例 > 60%
- Today 空状态用户点击"添加信息源"/"安装 Starter Pack"的比例 > 40%
- 无任何 TypeScript 编译错误

## §2 技术方案

### 2.1 数据库变更

**无数据库变更**。本版本为纯前端重构。

### 2.2 Rust 后端

**无 Rust 变更**。本版本不新增 IPC 命令，不修改后端模块。

> 注意：Today 页面当前使用已有的 `get_articles` HTTP API + `today` 过滤参数获取"今日文章"。v2.2 暂时保留这一数据获取方式，v2.3 引入 Pipeline 后切换到 `get_today_signals`。

### 2.3 前端

#### 2.3.1 路由变更

**修改**：`apps/desktop/src/config.ts`

```typescript
// 现有
LOCAL_TODAY = "/local/today"

// 保持不变，但在 index.tsx 中将其设为默认路由
```

**修改**：`apps/desktop/src/index.tsx`

将默认重定向从 `/local/all` 或 `/` 改为 `/local/today`。确保首次加载时自动导航到 Today 页面。

#### 2.3.2 新增 Store Slice

**创建**：`apps/desktop/src/stores/createTodaySlice.ts`

```typescript
interface TodayState {
  // 页面状态
  loading: boolean;
  error: string | null;

  // 今日文章（复用现有 Article 结构，后续 2.3 替换为 Signal[]）
  articles: Article[];
  lastUpdated: string | null;

  // Actions
  setLoading: (loading: boolean) => void;
  fetchTodayArticles: () => Promise<void>;
  clearError: () => void;
}
```

**修改**：`apps/desktop/src/stores/index.ts`

将 `createTodaySlice` 组合进 `useBearStore`。

#### 2.3.3 新增页面组件

**创建**：`apps/desktop/src/layout/Intelligence/` 目录

```
Intelligence/
├── TodayPage.tsx           # 替代 ArticleContainer 作为 /local/today 的页面组件
├── TodayEmptyState.tsx     # 空状态组件（无文章 / 无 API Key / Pipeline 未运行）
└── TodayArticleList.tsx    # 临时：文章列表展示（2.3 替换为 SignalList）
```

**`TodayPage.tsx`** 结构：

```
┌──────────────────────────────────────────────┐
│  Today                        ⚙ Settings     │
├──────────────────────────────────────────────┤
│                                              │
│  [空状态 / 文章列表]                          │
│                                              │
└──────────────────────────────────────────────┘
```

**`TodayEmptyState.tsx`** 处理三种空状态：

| 场景 | 显示文案 | 操作按钮 |
|------|---------|---------|
| 无订阅源 | "今天还没有新文章。添加信息源后，AI 将为你筛选最重要的内容。" | "添加信息源" / "安装 Starter Pack" |
| 有订阅源但无新文章 | "今天还没有新内容。信息源更新后，这里会自动刷新。" | 无 |
| 加载失败 | "加载失败，请稍后重试。" | "重试" |

#### 2.3.4 侧边栏导航调整

**修改**：侧边栏组件（`apps/desktop/src/components/MenuBar/` 或相关侧边栏文件）

- Today 菜单项提升到 Feeds 之前（或作为独立分组）
- Today 图标改为日历/信号图标以区分 Feeds
- 调整排序优先级
