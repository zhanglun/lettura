## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.4

**目标**：每条 Signal 都回答"为什么值得你看"。

**功能范围**：
- 设计 Why It Matters（WIM）模板：2–3 句话，≤80 words 英文 / ≤120 字中文
- 单条 WIM 只表达一个主判断，避免长摘要伪装成解释
- Signal 标题生成（聚类后文章组标题，≤12 words / ≤20 字中文）
- Pipeline 新增 WIM 生成步骤（在 Summary 之后、Ranking 之前）
- Pipeline 新增 Signal 标题生成步骤（在聚类之后、Summary 之后）
- 前端 SignalCard 新增 WIM 折叠区（默认折叠，点击展开）

**明确不做**：
- ❌ 来源透明 / 展开来源文章列表（2.5）
- ❌ 去重与压缩（2.6）
- ❌ 用户反馈闭环（2.7）
- ❌ Today Overview（2.8）
- ❌ Topic 对象（2.9+）

**验收标准**：
1. 每张 Signal 卡片默认显示折叠的 "💡 Why" 区域
2. 点击展开后显示 2–3 句话的 WIM 解释
3. WIM 表达一个具体判断，不是摘要复述
4. Signal 标题是聚类组标题（区别于单篇文章标题）
5. Pipeline 生成 WIM 失败时降级为 Summary 文本

**成功指标**：
- WIM 输出 90%+ 通过质量校验（2–4 句，≤100 words）
- WIM 展开率 > 50%（用户愿意看"为什么重要"）
- 无幻觉：WIM 内容可追溯到 Signal 的源文章

## §2 技术方案

### 2.1 数据库变更

**无新表 / 无新迁移**。`article_ai_analysis` 表在 v2.3 已创建，其中 `why_it_matters TEXT` 字段已预留。

- `article_ai_analysis.why_it_matters` — 存储 WIM 文本（v2.3 已预留字段，本版填充数据）

#### 2.2.2 Pipeline 流程变更

**修改**：`ai/pipeline.rs`

在 v2.3 Pipeline 基础上，新增两个步骤：

```
v2.3 Pipeline:
  SELECT unprocessed → embed → cluster → summary → rank → store

v2.4 Pipeline:
  SELECT unprocessed → embed → cluster → [NEW: signal_title]
    → summary → [NEW: why_it_matters] → rank → store
```

新增步骤详情：

1. **Signal 标题生成**（在聚类后）
   - 对每个聚类组（3-7 篇文章）
   - 收集组内文章标题
   - 调用 `generate_signal_title`
   - 批量大小 10，并发 2，超时 20s/batch

2. **Why It Matters 生成**（在 Summary 后，只对 Top N 文章）
   - 只对 relevance_score 排名前 10 的文章生成 WIM（不是全部）
   - 输入：summary + source_count + topic context
   - 调用 `generate_why_it_matters`
   - 批量大小 5，并发 2，超时 45s/batch
   - 失败降级：使用 Summary 文本作为 WIM

3. **Pipeline 进度事件新增阶段**
   - 新增 `"generating_wim"` 阶段（`pipeline:progress` 事件的 stage 字段）
   - 新增 `"generating_signal_titles"` 阶段

**无新 IPC 命令**。`get_today_signals` 返回的 `Signal` 结构在 v2.3 已包含 `why_it_matters` 字段（从 `article_ai_analysis.why_it_matters` 读取）。v2.4 只是开始填充这个字段。

WIM 区域交互：

| 状态 | 显示内容 | 视觉 |
|------|---------|------|

| expanded | 2-3 句 WIM 文本 | 向下滑出 transition 200ms ease-in-out |
| empty（WIM 降级为 Summary） | Summary 文本（与上方相同则隐藏 WIM 区域） | 次要色 |
| loading | Skeleton 行（2 行，宽度 70%） | 灰色脉冲 |
