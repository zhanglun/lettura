## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.13

**目标**：让 Topic 不是分类壳，而是真正的理解层。

**功能范围**：
- Pipeline 自动生成 Topic-level summary：当 Topic 包含 ≥3 篇文章且来自 ≥2 个来源时，调用 LLM 生成 `topics.description` 的增强描述（≤100 词）
- `get_topic_detail` 响应新增 `topic_summary` 字段（LLM 生成的 Topic 级总结，≤100 词）
- 文章按来源（Feed）分组展示：TopicDetailPage 增加来源分组视图，同来源文章折叠显示
- 多观点并置：当同一 Topic 下不同来源对同一事件有不同表述时，视觉上区分呈现
- Pipeline 更新已有 Topic 时重新生成/更新 `description`

**明确不做**：
- ❌ Topic 关注/取消关注（2.13）
- ❌ Topic 排序算法优化（2.13）
- ❌ 新增数据表
- ❌ 新增 IPC 命令
- ❌ Signal 检测 / 主动提醒（3.x）

**验收标准**：TopicDetailPage 展示 LLM 生成的 Topic summary，文章按来源分组显示，多来源观点可视觉区分。

**前置版本**：2.11（Topic 详情页 + topicSlice + 侧边栏入口）

**成功指标**：
- Topic 含 ≥3 篇文章且 ≥2 个来源时，`topic_summary` 非空
- 文章列表按来源分组，每组显示来源名和文章数
- 用户能通过 Topic summary 快速理解主题核心

## §2 技术方案

### 2.1 数据库变更

**无新增表**。本版本通过更新 `topics.description` 字段和丰富 `get_topic_detail` 响应来实现。

`topics` 表已有字段足够支撑：

| 字段 | 用途 |
|------|------|
| `description` | LLM 生成的 Topic 描述（2.10 已创建，2.12 增强生成逻辑） |
| `article_count` | 用于判断是否满足 summary 生成阈值 |
| `source_count` | 用于判断是否满足多来源要求 |

### 2.2 Rust 后端

**Pipeline 增强**（`ai/pipeline.rs` + `ai/topic.rs`）：

新增 Topic summary 生成步骤，在 Pipeline 聚类完成后触发：

```rust
// ai/topic.rs 新增函数
pub async fn generate_topic_summary(
    conn: &mut SqliteConnection,
    topic_id: i32,
    llm: &dyn LLMProvider,
) -> Result<()> {
    // 1. 查询 Topic 下的文章摘要列表
    // 2. 判断 article_count >= 3 && source_count >= 2
    // 3. 满足条件 → 调用 LLM 生成 summary（复用 ai-prompt-design.md §6 Prompt）
    // 4. UPDATE topics SET description = ? WHERE id = ?
    // 5. 不满足 → 跳过（保持现有 description）
}
```

**Topic summary Prompt**（参考 `prd/ai-prompt-design.md` §6，增强版）：

```
You are writing a comprehensive topic summary for a group of related articles.

Rules:
- Maximum 100 words in English, 150 characters in Chinese
- Summarize the core theme and key developments
- Note any diverging perspectives across sources
- Use factual language, no superlatives
- Present tense

Topic title: {topic_title}
Current description: {current_description}
Articles:
- {title_1} ({feed_1}): {summary_1}
- {title_2} ({feed_2}): {summary_2}
- {title_3} ({feed_3}): {summary_3}
...

Output the summary directly, no prefix, no quotes.
```

**`get_topic_detail` 响应增强**：

在现有返回结构基础上新增 `topic_summary` 字段：

```typescript
interface TopicDetail {
  // ...现有字段不变
  topic_summary?: string;       // 新增：LLM 生成的 Topic 级总结（≤100 词）
  // article_count >= 3 && source_count >= 2 时才有值
}
```

实现方式：`get_topic_detail` 查询时，读取 `topics.description` 作为 `topic_summary`（description 已由 Pipeline 更新为 LLM 生成的增强描述）。

**文章按来源分组**（`ai/topic.rs` 查询层）：

`get_topic_detail` 的 `articles` 查询增加来源分组信息：

```typescript
interface TopicDetail {
  // ...现有字段
  articles: TopicArticle[];           // 保持扁平列表（兼容）
  source_groups?: SourceGroup[];      // 新增：按来源分组
}

interface SourceGroup {
  feed_title: string;
  feed_uuid: string;
  article_count: number;
  articles: TopicArticle[];
}
```

**文件影响**：

| 文件 | 变更类型 | 说明 |
|------|---------|------|
| `ai/topic.rs` | 修改 | 新增 `generate_topic_summary` 函数，修改 `get_topic_detail` 查询增加 source_groups |
| `ai/pipeline.rs` | 修改 | 聚类步骤后调用 `generate_topic_summary` |
| `cmd.rs` | 修改 | `get_topic_detail` 返回结构增加 `topic_summary` 和 `source_groups` |

### 2.3 前端

**TopicDetailPage 增强**（`layout/Intelligence/Topics/TopicDetailPage.tsx`）：

```tsx
<div className="topic-detail">
  {/* 标题区（不变） */}
  <h1>{topic.title}</h1>

  {/* Topic Summary 区（新增） */}
  {topic.topic_summary && (
    <div className="topic-summary">
      <h3>📝 主题概要</h3>
      <p>{topic.topic_summary}</p>
    </div>
  )}

  {/* 统计区（不变） */}
  <div className="meta">...</div>

  {/* 文章列表 - 来源分组视图（增强） */}
  {topic.source_groups && topic.source_groups.length > 0 ? (
    <div className="source-groups">
      {topic.source_groups.map((group) => (
        <SourceGroup key={group.feed_uuid} group={group} />
      ))}
    </div>
  ) : (
    /* 降级：无分组信息时显示扁平列表 */
    <div className="article-list-flat">
      {topic.articles.map((article) => (
        <TopicArticleItem key={article.article_id} article={article} />
      ))}
    </div>
  )}
</div>
```

**新增组件**：`SourceGroup.tsx`

```tsx
// 按来源分组的文章展示

```

**多观点并置效果**：

当同一 Topic 下不同来源覆盖相同事件但有不同角度时：
- 同一来源的文章连续排列
- 来源组之间用分隔线区分
- 来源组标题旁显示来源名和文章数

**topicSlice 增强**（`stores/topicSlice.ts`）：
