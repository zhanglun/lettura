## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.8

**目标**：在 Today 列表上方给出一句话"今日信息判断"——让用户在进入具体 Signal 之前，先对今天的整体信息态势有一个快速感知。

**功能范围**：
- 一句话概览生成：基于今日 Top Signals 的标题和 summary，通过 LLM 生成一句话总览
- 概览格式约束：≤40 词（英文）/ ≤60 字（中文），提及 2-3 个最突出主题，客观冷静口吻
- 缓存机制：概览不存储在数据库，每次请求时实时生成，缓存 30 分钟（`is_stale` 标志）
- 前端展示：TodayOverview 组件位于 Today 页面顶部，支持 loading / loaded / stale / error / no_data / no_api_key 六种状态

**明确不做**：
- ❌ 概览历史记录（不存储到数据库）
- ❌ 用户自定义概览模板
- ❌ 概览编辑功能
- ❌ 多语言概览（跟随输入内容语言）

**验收标准**：Today 页面加载后 10 秒内顶部展示一句话概览，内容准确反映当日 Top Signals 的主要主题。超过 30 分钟后显示"X 分钟前更新"标签，用户可手动刷新。

**成功指标**：
- 概览生成延迟 < 10 秒（含 LLM 调用）
- 缓存命中率 > 60%（30 分钟内多次访问不重复调用 LLM）
- 概览内容与下方 Signals 一致性 > 90%（人工评审）

**核心逻辑**（`ai/overview.rs`）：

```rust
use std::sync::Mutex;
use std::time::{Duration, Instant};

/// 内存缓存，30 分钟过期
struct OverviewCache {
    overview: Option<TodayOverview>,
    generated_at: Option<Instant>,
    cache_ttl: Duration,  // 默认 30 分钟
}

struct TodayOverview {
    summary: String,           // 一句话概览
    signal_count: i32,         // 今日 Signal 总数
    article_count: i32,        // 今日新文章总数
    generated_at: String,      // ISO 8601 时间戳
    is_stale: bool,            // 是否超过缓存 TTL
}

/// 生成 Today Overview
/// 1. 从 DB 查询今日 Top Signals（标题 + summary）
/// 2. 检查缓存，未过期则直接返回
/// 3. 已过期或首次 → 调用 LLM 生成概览
/// 4. 写入内存缓存
async fn generate_today_overview(
    conn: &mut SqliteConnection,
    llm: &dyn LLMProvider,
    cache: &Mutex<OverviewCache>,
) -> Result<TodayOverview, String>;
```

**LLM Prompt**（参考 `prd/ai-prompt-design.md` §4）：

```
You are generating a daily intelligence briefing overview for a technical professional.

Given today's top signals, write ONE sentence that summarizes the overall information landscape.

Rules:
- Maximum 40 words in English, 60 characters in Chinese
- Mention the 2-3 most prominent themes
- Use concrete topic names, not "several topics"
- Do not use "today" or "in today's news" — the context is implied
- Tone: calm, informative, no urgency

Top signals:
{signal_1_title}: {signal_1_summary}
{signal_2_title}: {signal_2_summary}
...
{signal_n_title}: {signal_n_summary}

Output the overview directly, no prefix, no quotes.
```

**新增 IPC 命令**（`cmd.rs`）：

| 命令 | 签名 | 说明 |
|------|------|------|
| `get_today_overview` | `async fn() -> Result<TodayOverview, String>` | 获取今日概览（带缓存） |

> 参考：`prd/api-specification.md` §3.1（get_today_overview 接口定义）

**降级策略**（参考 `prd/ai-prompt-design.md` §7.2）：

- LLM 调用失败 → 返回 `"{article_count} 篇新文章，{signal_count} 条 Signal"` 作为 fallback
- 无文章数据 → 返回 `TODAY_NO_DATA` 错误
- API Key 未配置 → 返回 `AI_NO_API_KEY` 错误

**输出质量校验**（参考 `prd/ai-prompt-design.md` §7.3）：

```rust
fn validate_overview(text: &str) -> bool {
    let word_count = text.split_whitespace().count();
    word_count > 5 && word_count <= 50  // 留 20% 宽容度
}
```

校验失败则重试一次，仍然失败则使用降级文案。
