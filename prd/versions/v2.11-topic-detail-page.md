> **⚠️ 强制审查要求：开始执行本文档中的任何任务之前，必须先通读本文档，对照 `prd/` 目录下的相关 PRD 文档进行一次完整性审查。确认所有接口、数据模型、交互规格与最新 PRD 一致后，再开始编码。如发现不一致，先更新文档再动手。**

## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.11

**目标**：让用户第一次按主题而不是按文章理解信息。

**功能范围**：
- Topic 详情页完整组件 `/local/topics/:uuid`（TopicDetailPage），展示标题、描述、文章数、来源数、时间范围、近期变化
- TopicArticle 列表：文章标题、来源 Feed 名称、发布时间、相关性分数、50 字摘录
- Topics 加入侧边栏导航（位于 Today 之后、Feeds 之前）
- TopicListPage 升级：使用 topicSlice 管理状态，支持 TopicCard 交互
- 新增 Zustand slice：`topicSlice.ts`

**明确不做**：
- ❌ Topic 聚合/多来源归组（2.12）
- ❌ Topic-level summary / LLM 生成描述增强（2.12）
- ❌ Topic 排序算法优化（2.13）
- ❌ Topic 关注/取消关注功能（2.13）
- ❌ 新增 IPC 命令（复用 v2.10 的 `get_topics` + `get_topic_detail`）

**验收标准**：用户可通过侧边栏进入 Topics 列表页，点击 TopicCard 进入详情页，查看标题、描述、关联文章列表；文章按相关性排序显示。

**前置版本**：2.10（Topic 数据表 + IPC 命令 + SignalCard badge + 路由骨架）

**成功指标**：
- 侧边栏 Topics 入口点击后正确导航到 `/local/topics`
- TopicDetailPage 展示完整 Topic 信息和关联文章
- TopicArticle 列表按 `relevance_score` 降序排列

## §2 技术方案

### 2.1 数据库变更

**无新增表**。复用 v2.10 创建的 `topics` + `topic_articles` 表。

### 2.2 Rust 后端

**无新增 IPC 命令**。完全复用 v2.10 的 `get_topics` 和 `get_topic_detail`。

`get_topic_detail` 已返回完整结构（`prd/api-specification.md` §5.2）：

```typescript
interface TopicDetail {
  id: number;
  uuid: UUID;
  title: string;
  description: string;
  status: "active" | "archived" | "merged";
  article_count: number;
  source_count: number;
  first_seen_at: Timestamp;
  last_updated_at: Timestamp;
  is_following: boolean;       // v2.11 仍固定返回 false
  recent_changes?: string;     // v2.11 暂返回 null（2.12 实现）
  articles: TopicArticle[];
}

interface TopicArticle {
  article_id: number;
  title: string;
  link: string;
  feed_title: string;
  pub_date: Timestamp;
  relevance_score: number;
  excerpt?: string;            // 50 字摘录
}
```

### 2.3 前端

**新增 Zustand Slice**：`stores/topicSlice.ts`

```typescript
interface TopicState {
  topics: TopicItem[];         // Topic 列表
  selectedTopic: TopicDetail | null;  // 当前查看的 Topic 详情
  loading: boolean;            // 列表加载状态
  detailLoading: boolean;      // 详情加载状态
  error: string | null;        // 错误信息

  // Actions
  fetchTopics: (status?: string, sort?: string) => Promise<void>;
  fetchTopicDetail: (topicId: number) => Promise<void>;
  clearSelectedTopic: () => void;
}
```

在 `stores/index.ts` 中组合：

```typescript
import { createTopicSlice } from './topicSlice';

export const useBearStore = create<StoreState>()(

      ...createTopicSlice(...a),  // 新增

  )
);
```

**路由变更**（`config.ts` + `index.tsx`）：


**TopicDetailPage 路由参数处理**：

```typescript
// TopicDetailPage.tsx
const { uuid } = useParams<{ uuid: string }>();

// 需要先通过 get_topics 找到 uuid 对应的 topic.id，
// 或者后端新增按 uuid 查询的支持（建议在 get_topic_detail 中支持 uuid 参数）
```

> **注意**：当前 `get_topic_detail` 接受 `topicId: number`。若前端路由使用 `:uuid`，有两种方案：
> 1. 前端先从 topics 列表中匹配 uuid → 获取 id → 再调 `get_topic_detail(id)`
> 2. 后端 `get_topic_detail` 同时支持 `topic_id: number` 和 `topic_uuid: string`
>
> 推荐方案 1（无后端变更），TopicListPage 已加载所有 Topic 的 uuid↔id 映射。
