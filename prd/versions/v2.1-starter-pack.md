> **⚠️ 强制审查要求：开始执行本文档中的任何任务之前，必须先通读本文档，对照 `prd/` 目录下的相关 PRD 文档进行一次完整性审查。确认所有接口、数据模型、交互规格与最新 PRD 一致后，再开始编码。如发现不一致，先更新文档再动手。**

## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.1

**目标**：确保用户首次打开产品就有高质量数据可用，为后续所有 AI 能力提供数据基础。

**功能范围**：
- 8 个领域 Pack（AI、Developer、Startup、Product、Design、Science、Business、Tech News），每个 Pack 含 15–18 个高质量源
- Pack 数据结构：静态 JSON 随应用分发，安装后变为 `sources` + `feeds` 表记录
- 首次启动 Onboarding 流程（选择兴趣 → 安装 Pack → 异步抓取 → 进入 Today）
- 3 个 IPC 命令：`get_starter_packs` / `preview_pack` / `install_pack`
- 安装后自动触发首次抓取，通过 Tauri event `feed:sync_progress` 通知进度
- `onboarding_completed` 标记存 `lettura.toml` 的 `[app]` 节

**明确不做**：
- ❌ AI Pipeline / Today Intelligence（2.3+）
- ❌ 在线 Pack 更新 / 远程 Pack 源
- ❌ OPML 导入流程（本文档只预留接口，实际实现在 Onboarding Step 1 的"导入 OPML"按钮中触发现有 `import_opml` 命令）

**验收标准**：用户首次安装 → 选择兴趣 Pack → 3 分钟内 Today 页面展示首批抓取内容，且内容来自高质量源。

**成功指标**：
- Onboarding 完成率 > 80%（安装了至少 1 个 Pack）
- 首次安装到内容展示 < 3 分钟
- Pack 安装后源抓取成功率 > 90%

## §2 技术方案

### 2.1 数据库变更

**新增 `sources` 表**（迁移 `2026-05-01-120000_add_sources_table`）：

```sql
CREATE TABLE sources (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    uuid TEXT NOT NULL UNIQUE,
    feed_url TEXT NOT NULL,
    title TEXT,
    site_url TEXT,
    source_type TEXT NOT NULL CHECK(source_type IN ('starter_pack', 'user', 'opml_import')),
    pack_id TEXT,
    language TEXT NOT NULL DEFAULT 'en',
    quality_score REAL NOT NULL DEFAULT 0.5,
    weight REAL NOT NULL DEFAULT 1.0,
    is_active BOOLEAN NOT NULL DEFAULT 1,
    create_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_sources_source_type ON sources(source_type);
CREATE INDEX idx_sources_pack_id ON sources(pack_id);
CREATE INDEX idx_sources_feed_url ON sources(feed_url);
```

**修改 `feeds` 表**（同一迁移中）：

```sql
ALTER TABLE feeds ADD COLUMN source_id INTEGER REFERENCES sources(id) ON DELETE SET NULL;
CREATE INDEX idx_feeds_source_id ON feeds(source_id);
```

### 2.2 Rust 后端

**新增模块**：`apps/desktop/src-tauri/src/sources/`

```
sources/
├── mod.rs              # 模块入口
├── starter_pack.rs     # Pack 定义 / JSON 加载 / 安装逻辑
├── source_service.rs   # Source CRUD
└── models.rs           # Diesel 模型（Source, NewSource）
```

**新增 IPC 命令**（`cmd.rs`）：

| 命令 | 签名 | 说明 |
|------|------|------|
| `get_starter_packs` | `fn() -> Result<Vec<StarterPack>, String>` | 返回所有 Pack 列表（不含源详情） |
| `preview_pack` | `fn(pack_id: String) -> Result<PackPreview, String>` | 返回 Pack 完整内容（含源列表） |
| `install_pack` | `async fn(pack_ids: Vec<String>) -> Result<InstallResult, String>` | 批量安装 Pack，创建 sources+feeds，触发抓取 |

> **命令参数 serde 说明**：Tauri v2 IPC 自动处理 camelCase ↔ snake_case 转换。前端传 `{ packId }` → Rust 收到 `pack_id`。与现有 `import_opml` 的 `opml_content` ↔ `opmlContent` 模式一致，无需额外 serde 属性。

**配置变更**（`lettura.toml`）：

新增嵌套的 `AppConfig` 结构体，在 `lettura.toml` 中为 `[app]` section：

```rust
// core/config.rs 新增
#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct AppConfig {
    #[serde(default)]
    pub onboarding_completed: bool,
}

// UserConfig 中新增字段
pub struct UserConfig {
    // ... existing fields ...
    #[serde(default)]
    pub app: AppConfig,
}
```

对应 TOML：

```toml
[app]
onboarding_completed = false

# 预留 2.3
# [ai]
# api_key = ""
# model = "gpt-4o-mini"
```

**Pack JSON 存储位置**：`apps/desktop/src-tauri/src/sources/packs/`（8 个 JSON 文件，随编译打入二进制 `include_str!` 或运行时从目录读取）

**Pack JSON 格式**（`icon` 使用 lucide 图标名，`source_count` 动态计算不在 JSON 中存储）：

```json
{
  "id": "ai",
  "name": "AI & Machine Learning",
  "description": "AI research, breakthroughs, and industry trends",
  "icon": "brain",
  "language": "en",
  "tags": ["artificial-intelligence", "llm", "tools"],
  "sources": [
    {
      "feed_url": "https://openai.com/news/rss.xml",
      "title": "OpenAI Blog",
      "site_url": "https://openai.com",
      "language": "en"
    }
  ]
}
```

8 个 Pack 对应图标：AI=`brain`, Developer=`code-2`, Startup=`rocket`, Product=`package`, Design=`palette`, Science=`flask-conical`, Business=`briefcase`, Tech News=`newspaper`。

**文件影响**：

| 文件 | 变更类型 |
|------|---------|
| `cmd.rs` | 新增 3 个 `#[command]` 函数 |
| `lib.rs` | 注册新命令到 `invoke_handler` |
| `main.rs` | 可能需要初始化 sources 模块 |
| `schema.rs` | `diesel print-schema` 自动更新 |
| `core/config.rs` | 新增 `[app]` 配置节读取 |
| `feed/channel.rs` | `install_pack` 调用现有 `sync_feed` 逻辑 |

### 2.3 前端

**新增 Store Slice**：`apps/desktop/src/stores/onboardingSlice.ts`

```typescript
interface OnboardingState {
  step: 1 | 2 | 3 | 4;
  packs: StarterPack[];
  selectedPackIds: string[];
  installStatus: {
    total_feeds: number;
    completed_feeds: number;
    failed_feeds: number;
    articles_fetched: number;
  };
  isInstalling: boolean;
}
```

**新增组件目录**：`apps/desktop/src/layout/Onboarding/`

```
Onboarding/
├── OnboardingDialog.tsx    # 容器组件（全屏 Dialog）
├── WelcomeStep.tsx         # Step 1: 欢迎
├── SelectPackStep.tsx      # Step 2: 选择 Pack
├── InstallingStep.tsx      # Step 3: 安装进度
└── CompleteStep.tsx        # Step 4: 完成
```

**路由守卫**（`apps/desktop/src/App.tsx`）：

```typescript
const needsOnboarding = !userConfig.onboarding_completed && feeds.length === 0;
if (needsOnboarding) { /* 显示 OnboardingDialog */ }
```

**dataAgent.ts 扩展**（`apps/desktop/src/helpers/dataAgent.ts`）：

```typescript
const PACK_COMMANDS = {
  getStarterPacks: () => invoke('get_starter_packs'),
  previewPack: (id: string) => invoke('preview_pack', { packId: id }),
  installPacks: (ids: string[]) => invoke('install_pack', { packIds: ids }),
  importOpmlAsSource: (opmlContent: string) => invoke('import_opml_as_source', { opmlContent }),
};
```

**确认安装弹窗**（内嵌在 SelectPackStep 中）：

点击"继续"按钮后，弹出 Radix `AlertDialog` 显示确认摘要：
- 列出已选 Pack 名称及源数量
- 底部统计"共 N 个信息源"
- "返回"按钮关闭弹窗回到选择，"确认安装"按钮触发 `installPacks()` 并进入 Step 3

**OPML 导入**（WelcomeStep 中独立实现，不依赖现有 ImportAndExport）：

点击"导入 OPML"链接 → `@tauri-apps/plugin-dialog` 的 `open()` 选文件 → `readTextFile()` 读取内容 → 调用 `import_opml_as_source` 命令（创建 sources + feeds，source_type='opml_import'）→ 触发首次抓取 → 进入 Today

**事件监听**：

```typescript
listen('feed:sync_progress', (event) => { /* 更新 Onboarding 安装进度 */ });
```

**i18n 新增 key**：`prd/frontend-interaction-spec.md` §5.1 中 `onboarding.*` 部分

### 2.4 依赖

**无新增依赖**。此版本只使用现有 crate（diesel, tauri, serde, serde_json, uuid）和现有前端库。
