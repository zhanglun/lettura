## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.7

**目标**：形成最小用户偏好输入——让用户通过简单的反馈动作告诉系统哪些 Signal 有价值、哪些不相关、哪些值得持续关注。

**功能范围**：
- 三种反馈类型：有用（useful）、不相关（not_relevant）、继续关注（follow_topic）
- 反馈元数据记录：反馈类型、时间、关联 Signal ID、可选文字评论
- 反馈后视觉响应：按钮高亮、toast 通知、其他按钮禁用
- 反馈影响排序：useful → relevance_score +0.1，not_relevant → relevance_score -0.2，follow_topic → relevance_score +0.15（初始权重，可调）

**明确不做**：
- ❌ 基于反馈的模型训练 / 个性化推荐算法
- ❌ 反馈修改（提交后不可更改）
- ❌ 反馈删除（2.x 阶段不做）
- ❌ 复杂的兴趣模型（3.x 阶段）

**验收标准**：用户点击 SignalCard 上的反馈按钮 → 按钮高亮 + toast 提示 → 数据库 `user_feedback` 表写入记录 → 对应 Signal 的 `relevance_score` 实时更新 → 刷新 Today 后排序反映反馈权重。

**成功指标**：
- 反馈交互延迟 < 200ms（按钮响应）
- 反馈记录写入成功率 > 99.9%
- 用户 7 日内反馈率 > 30%（至少对 1 条 Signal 给出反馈）

```sql
CREATE TABLE user_feedback (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    signal_id INTEGER NOT NULL REFERENCES article_ai_analysis(id) ON DELETE CASCADE,
    feedback_type TEXT NOT NULL CHECK(feedback_type IN ('useful', 'not_relevant', 'follow_topic')),

CREATE INDEX idx_feedback_created ON user_feedback(create_date);
```

**反馈权重配置**（初始值，后续可调）：

```rust
/// 反馈对 relevance_score 的影响权重（可调）
const FEEDBACK_WEIGHT_USEFUL: f32 = 0.1;
const FEEDBACK_WEIGHT_NOT_RELEVANT: f32 = -0.2;
const FEEDBACK_WEIGHT_FOLLOW_TOPIC: f32 = 0.15;

/// 聚合同一 Signal 的所有反馈，计算累计权重
fn calculate_feedback_score_adjustment(feedbacks: &[UserFeedback]) -> f32 {
    feedbacks.iter().map(|fb| match fb.feedback_type.as_str() {
        "useful" => FEEDBACK_WEIGHT_USEFUL,
        "not_relevant" => FEEDBACK_WEIGHT_NOT_RELEVANT,
        "follow_topic" => FEEDBACK_WEIGHT_FOLLOW_TOPIC,
        _ => 0.0,
    }).sum()
}
```

**新增 IPC 命令**（`cmd.rs`）：

| 命令 | 签名 | 说明 |
|------|------|------|
| `submit_feedback` | `fn(signal_id: i32, feedback_type: String, comment: Option<String>) -> Result<FeedbackResult, String>` | 提交反馈并更新 relevance_score |
| `get_feedback_history` | `fn(limit: Option<i32>, offset: Option<i32>) -> Result<Vec<FeedbackEntry>, String>` | 获取反馈历史（Settings 展示用） |

```rust
#[derive(Serialize)]
struct FeedbackResult {
    recorded: bool,               // 是否成功记录
    updated_score: Option<f32>,   // 更新后的 relevance_score
}

#[derive(Serialize)]
struct FeedbackEntry {
    id: i32,
    signal_id: i32,
    signal_title: String,         // 关联查询 article_ai_analysis → articles.title
    feedback_type: String,
    created_at: String,           // ISO 8601
}
```

**submit_feedback 执行流程**：

1. 校验 `feedback_type` 是否为 `useful` / `not_relevant` / `follow_topic` 之一
2. 校验 `signal_id` 是否存在于 `article_ai_analysis` 表
3. INSERT `user_feedback` 记录
4. 查询该 Signal 的所有历史反馈，计算累计权重
5. UPDATE `article_ai_analysis.relevance_score += 累计权重`（clamp 到 [0.0, 1.0]）
6. 返回 `FeedbackResult { recorded: true, updated_score: Some(new_score) }`

### 2.3 前端

**SignalCard 反馈按钮**（`layout/Intelligence/SignalCard.tsx`）：

三个反馈按钮放在 SignalCard 底部：

```
┌──────────────────────────────────────┐
│ ... SignalCard 内容 ...              │
│                                      │
│  👍 有用   👎 不相关   📌 继续关注   │  ← 反馈按钮行
└──────────────────────────────────────┘
```

交互规则：
- 点击按钮 → 调用 `submit_feedback(signalId, type, null)`
- 成功后：被点击按钮变为实心高亮，其余按钮禁用（`pointer-events: none`, `opacity: 0.5`）
- Toast 通知：useful → "已标记为有用"，not_relevant → "已标记为不相关"，follow_topic → "已加入关注"
- 失败后：Toast 错误提示，按钮恢复可点击状态

**TodaySlice 扩展**（`stores/todaySlice.ts`）：

```typescript
interface TodayState {
  // ...existing
  feedbackMap: Record<number, FeedbackType | null>;  // signalId → feedbackType
}

type FeedbackType = 'useful' | 'not_relevant' | 'follow_topic';
```

**dataAgent.ts 扩展**：

```typescript
const FEEDBACK_COMMANDS = {
  submitFeedback: (signalId: number, feedbackType: FeedbackType, comment?: string) =>
    invoke('submit_feedback', { signalId, feedbackType, comment: comment ?? null }),
```
