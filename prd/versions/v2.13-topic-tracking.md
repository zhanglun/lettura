## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.13

**目标**：让 Topic 成为稳定的理解入口。

**功能范围**：
- Topic 关注/取消关注：新增 `follow_topic` / `unfollow_topic` IPC 命令
- Topic 排序算法：relevance score = f(article_count, source_count, recent_change_rate, user_feedback_weight)
- TopicListPage 支持多种排序（相关性、最近更新、文章数量）和筛选（全部 / 已关注）
- TopicCard 和 TopicDetailPage 新增关注/取消关注按钮
- **2.x 收口版本**：包含 Topic 使用验证指标、用户反馈分析框架、判断 Signal/Ask 是否进入 3.x 的决策清单

**明确不做**：
- ❌ Signal 独立对象（3.x）
- ❌ Ask 查询入口（3.x）
- ❌ 主动提醒/推送（3.x）
- ❌ Topic 时间线/历史演化（3.x）
- ❌ 长期记忆/报告（3.x）

**验收标准**：
1. 用户可关注/取消关注 Topic，状态在 TopicCard 和 TopicDetailPage 一致
2. Topic 列表支持按相关性/最近/文章数排序
3. "已关注"筛选 Tab 只显示关注的 Topic
4. 2.x 收口验证清单全部完成

**前置版本**：2.12（Topic 聚合 + Topic summary + 来源分组）

**成功指标**：
- 用户关注 Topic 后，TopicCard 和详情页状态一致
- Topic 排序反映文章数、来源数、近期活跃度和用户反馈权重
- 2.x 整体验证通过，可决策是否进入 3.x

## §2 技术方案

### 2.1 数据库变更

**方案决策**：使用 `topic_follows` 独立表（而非在 `topics` 表加 `is_following` 列），因为关注是用户行为，应独立于 Topic 本身的属性。

**新增 `topic_follows` 表**（迁移 `2026-06-01-000007_create_topic_follows`）：

```sql
-- up.sql
CREATE TABLE topic_follows (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    topic_id INTEGER NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
    followed_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(topic_id)
);

CREATE INDEX idx_topic_follows_topic ON topic_follows(topic_id);
```

```sql
-- down.sql
DROP TABLE IF EXISTS topic_follows;
```

### 2.2 Rust 后端

**Topic 排序算法**（`ai/ranking.rs` 新增 Topic 排序函数）：

```rust
/// 计算 Topic 的综合相关性分数
/// score = w1 * norm(article_count)
///       + w2 * norm(source_count)
///       + w3 * norm(recent_change_rate)
///       + w4 * norm(user_feedback_weight)
///       + w5 * is_following_bonus
pub fn compute_topic_relevance(topic: &TopicWithMetrics) -> f64 {
    let w1 = 0.25;  // 文章数量
    let w2 = 0.20;  // 来源多样性
    let w3 = 0.30;  // 近期活跃度（last 24h 新增文章占比）
    let w4 = 0.15;  // 用户反馈权重（来自 user_feedback）
    let w5 = 0.10;  // 关注加成

    let article_score = normalize(topic.article_count, 1.0, 50.0);
    let source_score = normalize(topic.source_count, 1.0, 20.0);
    let recency_score = topic.recent_change_rate; // 最近24h新增文章数 / 总文章数
    let feedback_score = topic.avg_feedback_score; // 该Topic下文章的平均feedback分数
    let follow_bonus = if topic.is_following { 1.0 } else { 0.0 };

    (w1 * article_score + w2 * source_score + w3 * recency_score
     + w4 * feedback_score + w5 * follow_bonus).min(1.0)
}

fn normalize(value: f64, min: f64, max: f64) -> f64 {
    ((value - min) / (max - min)).clamp(0.0, 1.0)
}
```

**`get_topics` 增强**：返回时附加 `is_following` 和 `relevance_score`。

`get_topics` 现有参数（`prd/api-specification.md` §5.1）：

```typescript
interface GetTopicsParams {
  status?: "active" | "archived" | "all";       // 默认 "active"
  sort?: "relevance" | "recent" | "article_count";  // 默认 "relevance"
  limit?: number;                                // 默认 20，最大 50
}
```

排序实现：
- `relevance`：使用 `compute_topic_relevance` 的分数降序
- `recent`：`last_updated_at` DESC
- `article_count`：`article_count` DESC

**`get_topic_detail` 增强**：返回 `is_following` 实际查询 `topic_follows` 表（v2.10/2.11 固定返回 false，2.13 改为真实查询）。
