## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.11

**目标**：从"今日最重要"自然过渡到"这些内容属于同一件事"。

**功能范围**：
- 定义 Topic 最小对象（`topics` 表 + `topic_articles` 关联表）
- Pipeline 聚类步骤：当文章形成 cluster 时自动创建/更新 Topic，写入 `topic_articles`
- `article_ai_analysis` 表新增 `topic_id` 字段，将 Signal 关联到 Topic
- SignalCard 组件增加 Topic 标记：若 Signal 关联了 Topic，显示可点击的 Topic badge
- 新增 `/local/topics` 路由（TopicListPage），但暂不加入侧边栏导航，仅通过 SignalCard badge 链接可达
- 新增 2 个 IPC 命令：`get_topics` / `get_topic_detail`

**明确不做**：
- ❌ 侧边栏 Topics 入口（2.11）
- ❌ Topic 详情页完整组件（2.11）
- ❌ Topic 排序 / 关注 / 跟踪功能（2.12–2.13）
- ❌ Topic-level summary 生成（2.12）
- ❌ 新增前端 Zustand slice（2.11）

**验收标准**：Pipeline 运行后，聚类产生的文章组自动形成 Topic，SignalCard 上显示 Topic badge，点击 badge 可跳转到 `/local/topics` 页面并看到 Topic 列表。

**前置版本**：2.3（article_ai_analysis 表 + Pipeline 基础流程）、2.6（向量去重/聚类基础）

**成功指标**：
- Pipeline 运行后，`topics` 表有记录，`topic_articles` 关联正确
- SignalCard 上有 `topic_id` 的卡片显示 Topic badge
- Topic badge 点击后跳转到 `/local/topics`，列表可正常展示

## §2 技术方案

### 2.1 数据库变更

**新增 `topics` 表 + `topic_articles` 表**（迁移 `2026-06-01-000005_create_topics_tables`）：

```sql
-- up.sql
CREATE TABLE topics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    uuid TEXT NOT NULL UNIQUE,
    title TEXT NOT NULL,
    description TEXT,
    status TEXT NOT NULL DEFAULT 'active' CHECK(status IN ('active', 'archived', 'merged')),
    article_count INTEGER NOT NULL DEFAULT 0,
    source_count INTEGER NOT NULL DEFAULT 0,
    first_seen_at TIMESTAMP,
    last_updated_at TIMESTAMP,
    create_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_topics_status ON topics(status);
CREATE INDEX idx_topics_updated ON topics(last_updated_at);

CREATE TABLE topic_articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    topic_id INTEGER NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
    article_id INTEGER NOT NULL REFERENCES articles(id) ON DELETE CASCADE,
    relevance_score REAL,
    create_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(topic_id, article_id)
);

CREATE INDEX idx_topic_articles_topic ON topic_articles(topic_id);
CREATE INDEX idx_topic_articles_article ON topic_articles(article_id);
```

```sql
-- down.sql
DROP TABLE IF EXISTS topic_articles;
DROP TABLE IF EXISTS topics;
```

**`article_ai_analysis` 表已有 `topic_id` 字段**（2.3 迁移中已定义），本版本实际使用该字段。

**新增 IPC 命令**：

| 命令 | 签名 | 说明 |
|------|------|------|
| `get_topics` | `fn(status?: string, sort?: string, limit?: number) -> Result<Vec<TopicItem>, String>` | 返回 Topic 列表 |
| `get_topic_detail` | `fn(topic_id: number) -> Result<TopicDetail, String>` | 返回 Topic 详情 + 关联文章 |

**TopicItem 返回结构**（与 `prd/api-specification.md` §5.1 一致）：

```typescript
interface TopicItem {
  id: number;
  uuid: UUID;
  title: string;
  description: string;
  status: "active" | "archived" | "merged";
  article_count: number;
  source_count: number;
  first_seen_at: Timestamp;
  last_updated_at: Timestamp;
  is_following: boolean;  // v2.10 暂固定返回 false
}
```

**TopicDetail 返回结构**（与 `prd/api-specification.md` §5.2 一致）：

```typescript
interface TopicDetail {
  id: number;
  uuid: UUID;
  title: string;
  description: string;
  status: "active" | "archived" | "merged";
  article_count: number;
  source_count: number;
  first_seen_at: Timestamp;
  last_updated_at: Timestamp;
  is_following: boolean;
  recent_changes?: string;      // v2.10 暂返回 null
  articles: TopicArticle[];
}

interface TopicArticle {
  article_id: number;
  title: string;
  link: string;
  feed_title: string;
  pub_date: Timestamp;
  relevance_score: number;
  excerpt?: string;
}
```

**Pipeline 聚类步骤增强**（`ai/pipeline.rs`）：

```rust
// 伪代码：聚类后的 Topic 处理
fn assign_or_create_topic(conn: &mut SqliteConnection, cluster: &ArticleCluster) -> Result<i32> {
    // 1. 尝试匹配已有 Topic（基于 centroid 距离）
    // 2. 距离 > 阈值 → 创建新 Topic（INSERT topics，生成 uuid + title）
    // 3. 距离 ≤ 阈值 → 更新已有 Topic（UPDATE article_count, source_count, last_updated_at）
    // 4. INSERT topic_articles（每篇文章关联到 Topic）
    // 5. UPDATE article_ai_analysis SET topic_id = ? WHERE article_id IN (...)
    // 6. 返回 topic_id
}
```

**Topic 描述生成**（`ai/topic.rs` 调用 `ai/llm.rs`，复用 `prd/ai-prompt-design.md` §6 的 Topic 描述 Prompt）。

**错误码**：`TOPIC_NOT_FOUND`、`TOPIC_EMPTY`（与 `prd/api-specification.md` §1.2 一致）。

### 2.3 前端

**SignalCard 组件修改**（`layout/Intelligence/SignalCard.tsx`）：

```tsx
{/* 新增：Topic badge（条件渲染） */}
{signal.topic_id && signal.topic_title && (
  <Link
    to="/local/topics"
    className="topic-badge"
  >
    {signal.topic_title}
  </Link>
)}
```

- badge 样式：圆角胶囊形，次要色背景，12px 字号，可点击
- 位置：SignalCard 标题下方或来源计数区域旁

**`get_today_signals` 返回结构**（已有 `topic_id` 和 `topic_title` 字段，见 `prd/api-specification.md` §3.2 Signal 接口），本版本前端开始使用这两个字段。

**新增路由**：`/local/topics` → `TopicListPage`（最小骨架页面）

```tsx
// config.ts 新增
TOPICS: '/local/topics',

// 路由注册
<Route path="/local/topics" element={<TopicListPage />} />
```

**TopicListPage 骨架**（`layout/Intelligence/TopicListPage.tsx`）：

```tsx
// 最小实现：直接调用 get_topics IPC，展示 TopicItem 列表
// 不引入独立 topicSlice，直接在组件内用 useState + useEffect
const [topics, setTopics] = useState<TopicItem[]>([]);
const [loading, setLoading] = useState(true);

useEffect(() => {
  invoke<TopicItem[]>('get_topics', { status: 'active', sort: 'recent', limit: 20 })
    .then(setTopics)
    .finally(() => setLoading(false));
}, []);
```

**暂不加入侧边栏**：2.10 的 Topic 列表页仅通过 SignalCard badge 链接到达，不修改侧边栏导航。

