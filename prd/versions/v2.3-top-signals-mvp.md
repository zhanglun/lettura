## §1 需求摘要

**来源**：`prd/version-roadmap-v2.md` §3.3 · 2.3

**目标**：提供 3–5 条最值得关注的内容，而不是文章流。

**功能范围**：
- 定义 Signal 卡片最小字段（id, title, summary, relevance_score, source_count, sources, created_at）
- 控制展示数量（默认 5 条，上限 10 条）
- 约束标题长度（≤12 words 英文 / ≤20 字中文）与一句话结论长度（≤50 words 英文 / ≤80 字中文）
- 实现 AI Pipeline 最小版：Feed sync 完成后 `tokio::spawn` 异步处理
- Pipeline 流程：SELECT 未处理文章 → Embed batch 32 → store → 增量聚类 → LLM Summary batch 5 → rank → store
- EmbeddingProvider + LLMProvider trait 抽象，统一 async-openai 调用
- TOML `[ai]` 配置节（api_key, model, embedding_model, base_url）
- 4 个新 IPC 命令（get_today_signals, get_ai_config, save_ai_config, validate_ai_config）+ trigger_pipeline
- AI Config Dialog 在 Settings 页面
- Pipeline 状态指示器（idle/running/done/error）
- 前端监听 `pipeline:*` 事件

**明确不做**：
- ❌ Why It Matters（2.4）
- ❌ 来源透明 / 展开来源文章列表（2.5）
- ❌ sqlite-vec 集成（2.6，本版 Embedding 结果直接存 JSON 字段或简单余弦匹配）
- ❌ HDBSCAN 聚类（本版用简单 centroid 匹配）
- ❌ 用户反馈闭环（2.7）
- ❌ Today Overview 一句话概览（2.8）
- ❌ Topic 对象（2.9+）

**验收标准**：
1. Feed sync 完成后自动触发 Pipeline，后台处理不阻塞 UI
2. Pipeline 完成后前端 Today 页面展示 3–5 张 Signal 卡片
3. 每张 Signal 卡片显示：标题、一句话结论、来源数量
4. Settings 页面可配置 AI API Key，保存后持久化到 `lettura.toml`
5. Pipeline 运行时 Today 页面右上角显示状态指示器
6. API Key 未配置时显示引导而非静默失败

**成功指标**：
- Pipeline 单次处理 100 篇文章 < 3 分钟
- Signal 卡片标题无幻觉（可追溯到源文章）
- API 调用成本单次 < $0.025
- 首次配置 API Key 到看到 Signal < 5 分钟

**新增 `article_ai_analysis` 表**（迁移 `2026-05-01-000003_create_article_ai_analysis`）：

```sql
CREATE TABLE article_ai_analysis (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    article_id INTEGER NOT NULL REFERENCES articles(id) ON DELETE CASCADE,
    summary TEXT,
    why_it_matters TEXT,
    relevance_score REAL,
    topic_id INTEGER,

CREATE INDEX idx_ai_analysis_article ON article_ai_analysis(article_id);
```

```sql
CREATE TABLE pipeline_runs (
```

#### 2.2.3 Pipeline 流程

```rust
// ai/pipeline.rs
pub async fn run_pipeline(
    conn: &mut SqliteConnection,
    ai_config: &AiConfig,
    app_handle: &AppHandle,
) -> Result<PipelineResult, PipelineError> {
    // 1. 创建 pipeline_runs 记录（status=running）
    // 2. emit pipeline:started
    // 3. SELECT 未处理文章（article_ai_analysis 中无记录的）
    //    batch size: 32
    // 4. 调用 Embedding API（并发 3，每批 32）
    // 5. 存储向量（本版存 JSON，2.6 迁移到 sqlite-vec）
    // 6. 增量聚类（简单 centroid 匹配，距离阈值 0.3）
    // 7. 调用 LLM 生成 Summary（batch 5，并发 2）
    //    Prompt 模板见 prd/ai-prompt-design.md §2
    // 8. 计算 relevance_score（聚类距离 + 源权重 + 时效性）
    // 9. INSERT article_ai_analysis
    // 10. 更新 pipeline_runs（status=completed, finished_at）
    // 11. emit pipeline:completed
}
```

**触发时机**：
- Feed sync 完成后 `tokio::spawn`（不阻塞 sync 返回）
- 手动触发（Settings 页面的 "运行 Pipeline" 按钮）
- 定时触发（复用现有 scheduler，间隔可配，默认每小时）

#### 2.3.3 事件监听

```typescript
// App.tsx 或 TodayPage.tsx
listen('pipeline:started', (event) => {
  store.setState({ pipelineStatus: 'running' });
});

listen('pipeline:progress', (event) => {
  const { stage, current, total } = event.payload;
  store.setState({ pipelineStage: stage, pipelineProgress: current / total });
});

listen('pipeline:completed', (event) => {
  store.setState({ pipelineStatus: 'done' });
  store.getState().today.fetchSignals(); // 刷新 Signals
});

listen('pipeline:failed', (event) => {
  store.setState({ pipelineStatus: 'error' });
});
```
