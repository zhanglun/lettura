# Today 页面规范

> **⚠️ 审查前置要求：在开始执行本文档中的任何设计或开发任务之前，必须先完整审查本文件，并与 `prd/ui-ux-redesign-spec.md` 总纲及 `prd/versions/` 下相关版本文档逐项对照，确认一致性后再开始实施。**

---

## 1. 页面定位

> **Today 给判断，不给文章流。**

Today 是新版 Lettura 的绝对默认入口。它是一个 **轻判断页**，负责回答：

> 今天最值得注意的变化是什么？

它的首要任务不是"快速浏览"，也不是"快速进入深读"，而是 **快速判断**。

### 2.2 四层结构

| 层级 | 组件 | 职责 |
|------|------|------|
| 标题区 | TodayHeader | Today 标题 + 一句副标题 + 更新时间/来源数状态 pill |
| 概览层 | TodaySummary | 1-2 句话，回答"今天最值得注意的变化是什么" |
| 判断层 | SignalList → SignalCard × 3-5 | 每张卡片是一个判断单元 |
| 辅助层 | 卡片内展开 / 轻抽屉 | 来源证据、状态、下一步动作 |

## 3. TodaySummary 规范

### 3.1 内容要求

- 1-2 句话
- 只回答"今天最值得注意的变化是什么"
- 不写成长摘要，不做新闻播报
- 语气：冷静、陈述性、不夸张

### 3.2 示例

好的：
> "AI coding tools 正在从辅助补全转向自主 Agent 工作流，这对开发者工具链的选购有直接影响。"

不好的：
> "以下是今天所有重要的科技新闻摘要..."

### 3.3 空状态

当没有足够数据生成 Summary 时：
- 显示"正在分析今天的内容..."
- 附上分析进度状态（如"已分析 12/42 源"）

---

## 4. SignalCard 规范

详见 `prd/ui/component-language-spec.md` 中的 SignalCard 部分。

### 4.1 固定层级（不可颠倒）

1. **标题** — Signal 的核心判断句
2. **一句话判断** — 对标题的简短展开
3. **Why It Matters** — 为什么你该在意（默认折叠）
4. **元信息** — 来源数 / Topic / 时间 / 置信度
5. **操作区** — 反馈按钮

### 4.2 默认数量

- 3-5 张卡片
- 不做分页，用滚动加载
- 每张卡片之间的间距要足够，不拥挤

---

## 5. 交互规则

### 5.1 点击行为

| 点击目标 | 行为 |
|---------|------|
| 卡片主体 | 展开更多解释（Why It Matters + 证据摘要） |
| Why It Matters | 展开/折叠解释层 |
| 来源数 | 展开来源证据层 |
| 原文链接 | 进入阅读器（二级动作） |

### 5.2 默认路径

```
看判断 → 看解释 → 看证据 → 再决定是否读原文
```

### 5.3 深读克制

"阅读原文"不应成为首页的主引导。它应该收在来源层中，只给最可信、最核心的来源更明确的主入口。

---

## 6. 反馈按钮规范

### 6.1 三个动作

| 按钮 | 语义 | 系统行为 |
|------|------|---------|
| 有用 | 强化该类判断 | 提升类似内容权重 |
| 不相关 | 降低类似内容权重 | 降低类似内容排序 |
| 继续关注 | 升级为跟踪对象 | 创建/加入 Topic |

### 6.2 视觉要求

- 不能做成夸张反应按钮
- 更像轻量控制动作
- 用文字标签而非图标（线框中的 emoji 仅为示意，实际使用纯文字标签如 "有用 · 不相关 · 继续关注"）

---

## 7. Pipeline 状态产品化

### 7.1 用户需要看到的状态

| 产品状态 | 文案示例 |
|---------|---------|
| 更新时间 | "Updated 2h ago" |
| 来源覆盖 | "Based on 42 sources" |
| 分析中 | "Analyzing... 12/42 sources processed" |
| 待更新 | "Results may change as more sources are analyzed" |

### 7.2 用户不需要看到的状态

- Embedding completed
- Clustering completed
- Vector index built
- Pipeline stage X

---

## 8. 空状态与边界情况

| 情况 | 表现 |
|------|------|
| 首次使用，无数据 | 显示 Onboarding 引导 |
| 有源但分析未完成 | "正在分析..." + 进度 |
| 今日无 Signal | "Today looks quiet. Nothing stands out." + 建议查看 Topics |
| AI 未配置 | 提示配置 AI → 跳转 Settings |
| 网络断开 | 离线提示 + 缓存的最后一次结果 |

---

## 9. 目标体验

用户应该在这个页面里感觉：

- 信息被整理过
- 重点被提前标出
- 我有证据可以验证
- 我随时能深入，但不用一上来就陷进去

---

## 10. 与现有前端的关系

当前 Lettura 的 Today 页面（`src/pages/Today.tsx`）是基本文章列表。新版需要：

- 保留路由路径 `/today`
- 替换内容结构为上述四层
- 复用现有 Zustand store 的数据获取逻辑
- 新增 Today Summary / SignalCard 相关组件
- 不改动 Feeds 页面和其他页面的路由
