# 视觉系统规范

> **⚠️ 审查前置要求：在开始执行本文档中的任何设计或开发任务之前，必须先完整审查本文件，并与 `prd/ui-ux-redesign-spec.md` 总纲及 `prd/versions/` 下相关版本文档逐项对照，确认一致性后再开始实施。**

---

## 1. 整体视觉基调

亮色主基调，避免：
- 资讯网站式热闹
- 后台系统式僵硬
- 暗色终端式压迫

> **界面应该让人感觉"信息被整理好了"，而不是"设计在炫耀自己"。**

---

## 2. 配色系统

### 2.1 主色板

| 用途 | 色值（建议） | 说明 |
|------|-------------|------|
| 背景 | `#FAFAFA` / `#F8F9FA` | 轻灰白 |
| 面板 | `#FFFFFF` | 白色 |
| 面板（备选） | `#F5F5F5` | 极浅暖灰 |
| 边框 | `#E5E5E5` | 低对比细边框 |
| 文字（主） | `#1A1A1A` | 接近黑，不纯黑 |
| 文字（副） | `#6B7280` | 灰色，用于元信息 |
| 文字（辅助） | `#9CA3AF` | 浅灰，用于时间戳等 |

### 2.2 强调色

| 用途 | 色值（建议） | 说明 |
|------|-------------|------|
| 主强调 | `#2563EB` / `#3B82F6` | 偏冷蓝色体系 |
| 成功 | `#10B981` | 绿色 |
| 警告 | `#F59E0B` | 黄色 |
| 错误 | `#EF4444` | 红色 |
| 信息 | `#6366F1` | 靛蓝 |

### 2.3 不使用的颜色

- 不使用鲜艳的渐变色
- 不使用荧光色
- 不使用大面积深色
- 不使用品牌紫/品牌橙等高饱和色做背景

## 3. 字体

### 3.1 字体栈

| 用途 | 字体 |
|------|------|
| 英文 | system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif |
| 中文 | "PingFang SC", "Microsoft YaHei", sans-serif |
| 等宽 | "SF Mono", "Fira Code", monospace |

### 3.2 字号

| 用途 | 大小 | 说明 |
|------|------|------|
| 页面标题 | 24px / 1.5rem | Today / Topics / Feeds |
| 区块标题 | 18px / 1.125rem | Section headers |
| 卡片标题 | 16px / 1rem | SignalCard 标题 |
| 正文 | 14px / 0.875rem | 主要内容文字 |
| 辅助文字 | 12px / 0.75rem | 元信息、时间戳、pill |

### 3.3 字重

| 用途 | 字重 |
|------|------|
| 标题 | 600 (Semibold) |
| 正文 | 400 (Regular) |
| 辅助 | 400 (Regular) |
| 强调 | 500 (Medium) |

## 4. 间距与密度

### 4.1 基础间距单位

以 4px 为基础单位，所有间距为 4 的倍数：

| 用途 | 间距 |
|------|------|
| 组内间距 | 8px (2×) |
| 组间间距 | 16px (4×) |
| 区块间距 | 24px (6×) |
| 页面边距 | 24px (6×) |

### 4.2 页面密度等级

| 页面 | 密度 | 特点 |
|------|------|------|
| Today | 低密度 | 留白最多，卡片间距大 |
| Topics | 中密度 | 结构化但透气 |
| Feeds | 高密度 | 紧凑，强调信息量 |

---

## 5. 阴影与层级

### 5.1 阴影

| 层级 | 阴影 | 用途 |
|------|------|------|
| Level 0 | 无阴影 | 默认面板 |
| Level 1 | `0 1px 3px rgba(0,0,0,0.05)` | 卡片 |
| Level 2 | `0 4px 12px rgba(0,0,0,0.08)` | 浮起元素（drawer、tooltip） |

不使用 Level 3+ 的大面积阴影。

### 5.2 圆角

| 元素 | 圆角 |
|------|------|
| 卡片 | 8px |
| 按钮 | 6px |
| Pill / Badge | 9999px (全圆) |
| 输入框 | 6px |
| Drawer | 8px（左侧） |

## 6. 动画与过渡

### 6.1 原则

- 动画只用于辅助理解，不用于装饰
- 所有过渡时间 ≤ 200ms
- 使用 ease-out 曲线
- 尊重 prefers-reduced-motion

### 6.2 使用场景

| 场景 | 动画 |
|------|------|
| 卡片展开/折叠 | 高度变化，150ms ease-out |
| 页面切换 | 无动画（v2.x） |
| Hover 状态 | 背景色变化，100ms |
| Drawer 出现 | 右侧滑入，200ms ease-out |
| Tooltip 出现 | 淡入，100ms |

---

## 7. 暗色模式

### 7.1 当前阶段

v2.x 不实现暗色模式，但配色系统要预留暗色模式的设计空间。

### 7.2 预留原则

- 所有颜色使用 CSS 变量 / Tailwind token
- 不硬编码色值
- 预留 `dark` 前缀的 token

---

## 8. 页面视觉等级

| 页面 | 视觉特征 |
|------|---------|
| **Today 最轻** | 留白最多，强调判断，卡片最完整 |
| **Topics 最清** | 中等密度，更结构化，更像解释页 |
| **Feeds 最实** | 最工具化，密度可更高，更强调控制感 |

---

## 9. 与现有系统的关系

Lettura 当前使用：
- Tailwind v3 + Radix UI theme tokens（`tailwind.config.js`）
- shadcn-style 组件（`src/components/ui/`）
- `cn` helper（`src/helpers/cn.tsx`）

新版视觉系统应：
- 继续使用 Tailwind + shadcn 组件基础
- 新增上述配色/间距/字号 token
- 不引入新 UI 框架
- 复用现有 `cn` helper
