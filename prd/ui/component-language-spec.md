# 组件语言规范

> **⚠️ 审查前置要求：在开始执行本文档中的任何设计或开发任务之前，必须先完整审查本文件，并与 `prd/ui-ux-redesign-spec.md` 总纲及 `prd/versions/` 下相关版本文档逐项对照，确认一致性后再开始实施。**

---

## 1. 设计原则

整个产品围绕三种主组件语言建立统一性：

1. **Signal Card** — 判断、解释、证据、反馈的最小闭环
2. **Structured List** — 统一列表语言
3. **Lightweight Drawer / Inline Expand** — 证据层、补充层、状态层

信息层级原则（所有核心页面统一）：

> **主判断 → 解释 → 证据 → 动作**

只要这个顺序被破坏，产品就会回到 RSS Reader 的老逻辑。

---

## 2. Signal Card

### 2.1 定位

新版最核心组件。每张 SignalCard 只回答三件事：

1. 发生了什么
2. 为什么你该在意
3. 证据够不够支持这个判断

### 2.2 固定层级（不可颠倒）

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  🔵 标题                                                    │  ← Layer 1: 判断
│  最小模型达到 GPT-4 质量，成本大幅降低                        │
│                                                             │
│  💡 Why: 这改变了 embedding 密集型应用的经济性                │  ← Layer 2: 解释
│  ...                                                        │
│                                                             │
│  📰 5 articles · 3 sources · 🏷️ AI · ⏱ 2h ago             │  ← Layer 3: 证据
│                                                             │
│  有用  ·  不相关  ·  继续关注                                 │  ← Layer 4: 动作
│                                                             │
│  ▼ 展开来源                                                 │  ← 展开入口
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 层级定义

| 层级 | 内容 | 默认状态 |
|------|------|---------|
| Layer 1: 判断 | 标题 + 一句话判断 | 始终可见 |
| Layer 2: 解释 | Why It Matters | 默认折叠，可展开 |
| Layer 3: 证据 | 来源数 / Topic / 时间 / 置信度 | 始终可见（摘要） |
| Layer 4: 动作 | 反馈按钮 | 始终可见 |

### 2.4 展开行为

| 点击目标 | 展开内容 |
|---------|---------|
| 卡片主体 | Why It Matters + 证据摘要 |
| Why It Matters | 完整解释 |
| 来源数 | 来源证据列表 |
| 原文链接 | 阅读器 |

### 2.5 反馈按钮

| 按钮 | 语义 | 系统行为 |
|------|------|---------|
| 有用 | 强化该类判断 | 提升类似内容权重 |
| 不相关 | 降低类似内容权重 | 降低排序 |
| 继续关注 | 升级为跟踪对象 | 创建/加入 Topic |

视觉：文字标签，不做夸张反应图标。

### 2.6 组件接口（建议）

```typescript
interface SignalCardProps {
  id: string;
  title: string;                    // 判断标题
  summary: string;                  // 一句话判断
  whyItMatters: string;             // 解释
  sourceCount: number;              // 来源数
  articleCount: number;             // 文章数
  topic?: { id: string; name: string }; // 关联 Topic
  confidence: number;               // 置信度 0-1
  timestamp: string;                // 时间
  sources?: Source[];               // 展开后的来源列表
  onFeedback: (type: 'useful' | 'irrelevant' | 'follow') => void;
  onSourceClick: (sourceId: string) => void;
  onArticleClick: (articleId: string) => void;
}
```

## 3. Structured List

### 3.1 定位

统一列表语言，用于 Topics、Feeds、Search 结果。避免每页各做一套。

### 3.2 结构

```
┌─────────────────────────────────────────────────────────────┐
│  [Icon]  主信息                              [辅助信息]      │
│          副信息                                              │
│          元信息（来源数 · 时间 · 状态）                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 变体

| 变体 | 用于 | 特点 |
|------|------|------|
| TopicItem | Topics 列表 | 名称 + 定义 + 来源数 + 跟踪状态 |
| FeedItem | Feeds 列表 | 名称 + 健康状态 + 未读数 + 同步时间 |
| SearchResultItem | Search 结果 | 标题 + 摘要 + 来源 + 时间 |

### 3.4 交互

- Hover：浅灰背景
- Click：跳转详情
- 右键：上下文菜单（如适用）

## 4. Lightweight Drawer / Inline Expand

### 4.1 定位

用于证据层、补充层、状态层。减少切页，降低中断。

### 4.2 两种形式

#### A. Inline Expand

在卡片内部展开，不改变外部布局。

```
┌─────────────────────────────────────────────────────────────┐
│  Signal Card 主体                                           │
│  ...                                                        │
│  ▼ 展开来源                                                 │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Hacker News · "Article Title"                       │    │
│  │  The Verge · "Article Title"                         │    │
│  │  ...                                                 │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

#### B. Lightweight Drawer

从右侧滑入的轻抽屉，不占固定位置。

```
┌────────────────────────────────┬────────────────────────────┐
│                                │                            │
│  Main Content                  │  Evidence Drawer           │
│                                │  ─────────────             │
│                                │  Source 1                  │
│                                │  Source 2                  │
│                                │  ...                       │
│                                │                            │
│                                │  [×]                       │
└────────────────────────────────┴────────────────────────────┘
```

### 4.3 选择规则

| 场景 | 形式 |
|------|------|
| 少量证据（1-3 条） | Inline Expand |
| 较多证据（4+ 条） | Lightweight Drawer |
| 需要对比多条来源 | Lightweight Drawer |
| 快速确认一条来源 | Inline Expand |

## 5. 其他共用组件

### 5.1 Status Pill

用于展示系统状态的小标签：

```
Updated 2h ago · 42 sources active · 5 signals
```

- 圆角矩形
- 浅灰背景
- 小字号
- 出现在页面标题区或卡片元信息区

### 5.2 Confidence Indicator

用于表示判断的可信度：

- 不做显式百分比
- 用视觉暗示（颜色深浅 / 圆点大小 / 线条粗细）
- 低置信度时附加"证据有限"提示

### 5.3 Topic Tag

用于卡片中关联的 Topic：

- 可点击 → 跳转 Topic 详情
- 小标签样式
- 圆角 + 浅色背景
