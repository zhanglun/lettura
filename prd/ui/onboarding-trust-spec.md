# Onboarding 与信任建立规范

> **⚠️ 审查前置要求：在开始执行本文档中的任何设计或开发任务之前，必须先完整审查本文件，并与 `prd/ui-ux-redesign-spec.md` 总纲、`prd/onboarding-flow-spec.md` 及 `prd/versions/` 下相关版本文档逐项对照，确认一致性后再开始实施。**

---

## 1. Onboarding 的真正目标

不是教用户点按钮，而是 **重排预期**。

它要完成三件事：

1. 让用户明白：首页先看判断，不是先刷文章
2. 让用户相信：判断来自自己选的源
3. 让用户知道：自己仍然拥有控制权

---

## 2. 冷启动四步（语义重写）

### Step 1：欢迎

核心只讲一件事：

> Lettura 会先帮你判断今天什么重要。

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                     Lettura                                 │
│                                                             │
│         先判断，再阅读                                       │
│                                                             │
│  Lettura 不让你刷几百篇文章。                                │
│  它先帮你判断今天什么重要，再让你决定读什么。                  │
│                                                             │
│  ──────────────────────────────────────                      │
│                                                             │
│  工作方式：                                                  │
│  1. 选择你关注的信息领域                                     │
│  2. AI 自动分析每天最重要的变化                               │
│  3. 先看判断，再看原文                                       │
│                                                             │
│              [开始设置 →]                                    │
│                                                             │
│  已有 RSS 订阅？[导入 OPML]                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**与现有 `prd/onboarding-flow-spec.md` 的差异：**
- 副标题从"Your Personal Intelligence Feed"改为"先判断，再阅读"
- 工作方式第 3 步从"只看结论，不用刷几百篇文章"改为"先看判断，再看原文"
- 语气更冷静，减少营销感

### Step 2：选择 Pack

不要只展示"选源"，而要展示"你希望系统代表性观察什么"。

- 每个 Pack 的描述应该强调"观察领域"而非"内容分类"
- 例如：不写"AI & Machine Learning"，写"观察 AI 工具和模型的最新进展"

### Step 3：安装与分析

用户等待的不是"数据下载完成"，而是"判断工作台准备好"。

- 进度文案：不写"正在下载 42 个 RSS 源"，写"正在准备你的判断工作台..."
- 完成文案：不写"安装完成"，写"判断工作台已就绪"

### Step 4：进入 Today

完成页不是庆祝页，而是心智切换页。

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ✅ 准备就绪                                                │
│                                                             │
│  你的判断工作台已经准备好了。                                │
│                                                             │
│  接下来你会看到：                                            │
│  · Today 页面会展示今天最重要的变化                          │
│  · 每条判断都有来源可以验证                                  │
│  · 你的反馈会让判断越来越准确                                │
│                                                             │
│              [进入 Today →]                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 3. 信任建立的交互顺序

信任不是靠文案一次性建立的，而是靠前三次核心交互建立的：

### 3.1 三次信任触点

| 触点 | 用户行为 | 建立信任 |
|------|---------|---------|
| 第 1 次 | 阅读 Today Summary | 接受整体判断语气 |
| 第 2 次 | 展开 Why It Matters | 确认这个判断讲得通 |
| 第 3 次 | 查看来源证据 | 验证依据可信 |

这条链路必须顺畅。任何一个环节断裂，信任就会坍塌。

### 3.2 信任设计原则

- **判断要有依据**：每次展示判断时，来源证据必须可达
- **来源要透明**：来源名称 + 原文链接都要可追溯
- **反馈要有效**：用户的反馈要能明显改变后续结果
- **控制要明确**：用户要能随时调整源、模型、配置

## 4. 第一周使用路径

如果不设计首周路径，用户会自动退回旧习惯：找 Feed、点文章、开始刷。

### 4.1 Day 1：完成闭环

引导用户完成一条完整闭环：

```
Today Summary → Signal → Why → 来源 → 原文
```

这个闭环的核心是让用户感觉到：

> "我不用刷 100 篇文章，就能知道今天最重要的 3 件事。"

### 4.2 Day 2-3：感受反馈

让用户明确感受到：**反馈会改变排序结果**。

- 用户标记"不相关"后，后续 Today 中类似内容减少
- 用户标记"有用"后，类似内容权重提升
- 这个变化要让用户明显感知到

### 4.3 Day 4-7：进入 Topics

把用户自然带进 Topics：

- Today 中的 Signal 关联 Topic 标签
- 点击 Topic 标签进入 Topic 详情
- 从一次判断进入持续理解

## 5. 与现有 Onboarding 文档的关系

本规范是 `prd/onboarding-flow-spec.md` 的语义重写版。

- 技术实现（触发条件、TOML 标记、Pack 安装流程）沿用现有文档
- 文案和交互顺序按本规范重写
- 两者有冲突时，以本规范为准（后定义优先）
- 具体状态、接口、失败处理和验收清单见 `prd/ui/onboarding-interaction-spec.md`

**已知具体冲突点：**

| 冲突项 | `prd/onboarding-flow-spec.md` 原文 | 本规范改写 | 原因 |
|--------|-----------------------------------|-----------|------|
| 欢迎页副标题 | "Your Personal Intelligence Feed" | "先判断，再阅读" | 更直接传达产品核心价值，减少营销感 |
| 工作方式第 3 步 | "只看结论，不用刷几百篇文章" | "先看判断，再看原文" | 减少否定语气，改为正面描述 |
| Step 3 进度文案 | "正在下载 N 个 RSS 源..." | "正在准备你的判断工作台..." | 语义重定义：用户等待的是判断能力，不是数据下载 |
| Step 3 完成文案 | "安装完成" | "判断工作台已就绪" | 完成页是心智切换，不是庆祝 |
| Step 4 完成页 | 简单的"开始使用" | 详细的三条预期引导 | 帮助用户理解新版正确的工作方式 |
