## 1. 页面定位

> **Settings 给信任，不给杂项堆叠。**

Settings 不能再像杂项仓库。在新版里，它承担的其实是"信任建立"职责。

## 3. AI 配置

### 3.1 内容

| 配置项 | 类型 | 说明 |
|--------|------|------|
| API Key | password input | 用户的 AI API Key |
| Base URL | text input | API 端点（默认 OpenAI） |
| Model | select | 模型选择 |
| 当前状态 | status | 已连接 / 未配置 / 错误 |
| 最近分析 | info | 最近一次分析的时间和结果概况 |

### 3.2 信任建立

AI 配置页的核心目标不是"填表"，而是让用户理解：

- 我的 AI 配置只影响本地分析
- 我的 API Key 存储在本地
- 我可以随时更换模型
- 系统现在正在做什么

## 4. Sources 管理

### 4.1 内容

| 功能 | 说明 |
|------|------|
| Starter Pack | 安装/卸载 Pack |
| 已安装源 | 列表 + 健康状态 |
| 失效源 | 标记 + 处理建议 |
| 健康状态 | 同步频率 + 成功率 |
| 同步策略 | 频率 + 超时设置 |

### 4.2 与 Feeds 的区别

- Feeds 页面负责"操作源"（添加/删除/排序）
- Settings 的 Sources 管理负责"配置源的行为"（同步策略、Pack 管理）

## 5. 外观与阅读

| 配置项 | 类型 | 默认值 |
|--------|------|--------|
| 主题 | toggle | 亮色 |
| 字号 | slider | 16px |
| 行高 | slider | 1.6 |
| 阅读器偏好 | select | 排版/紧凑/舒适 |
| 卡片密度 | select | 舒适/紧凑 |

---

## 6. 应用行为

| 配置项 | 类型 | 默认值 |
|--------|------|--------|
| 开机启动 | toggle | 关 |
| 后台同步 | toggle | 开 |
| 通知 | toggle | 关 |
| 缓存保留 | select | 30 天 |
| 数据保留 | select | 90 天 |

---

## 7. 对应版本任务

Settings 的重构分散在多个版本中：

- v2.1：AI 配置基础（API Key / Model）
- v2.7+：Sources 管理完善
- v2.x 后期：外观与阅读偏好完善
