优先级 1：Feeds 文章行 dimmed 状态

  • mockup：.art-row.dimmed { opacity: 0.28; pointer-events: none }，内联阅读器展开时其余行降对比度。
  • ArticleListVirtual 已持有 expandedArticleUuid，需向 ArticleItem 传入 isDimmed prop 并应用样式。
  • 改动量极小（约 10 行），视觉感知提升明显。

优先级 2：Search 结果关键词 <mark> 高亮

  • mockup：snippet 中关键词用 <mark> 包裹，琥珀背景（var(--amber-3)）+ 深琥珀文字（var(--amber-11)）。
  • 方案一：后端在 snippet 字段中直接返回带 <mark> 的 HTML，前端用 dangerouslySetInnerHTML 渲染（需 DOMPurify 过滤）。
  • 方案二：前端拿到搜索词和纯文本 snippet，用正则匹配插入高亮组件（不依赖后端改动）。
  • 先确认后端 /search 接口是否已返回 highlight 字段，再选方案。

优先级 3：Starred 文章卡笔记展示

  • mockup：有笔记的文章卡底部显示 .saved-note（靛蓝左边框 3px + var(--accent-a3) 背景，12px 行高 1.5）。
  • 需先确认 ArticleResItem 数据模型中是否有 user_note 字段，以及后端 /starred 接口是否返回该字段。
  • 如字段存在：在 Starred 文章卡组件中条件渲染笔记块即可。
  • 如字段不存在：需先评估是否要新增数据模型和 UI 入口（笔记编辑）。

优先级 4：Feeds 文章列表 section label

  • mockup：文章行上方有粘性分组标签"未读 · 156 篇"（art-section-label，10px 大写灰底，position: sticky; top: 0）。
  • 当前未读数只显示在列表头，列表内无粘性分组。
  • 可在 ArticleListVirtual 的虚拟列表中插入一条非文章的 label 行，或在滚动容器顶部固定渲染。

优先级 5：Topics 详情面板补全

  • "Start Here 推荐阅读"：topic-start-card 需显示推荐理由 badge（"推荐先读"/"看反方观点"），当前仅有占位文案。
  • "状态说明"底部卡：topic-state-note，灰底小字"AI 每次运行后自动更新，你的追踪让它更准确"，当前 TopicDetailPanel 中缺失。
