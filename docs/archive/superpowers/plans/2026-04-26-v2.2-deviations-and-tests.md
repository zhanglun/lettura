## Task 1: Fix TodayPage conditional rendering (Deviation #3 — logic bug)

**Files:**
- Modify: `src/layout/Intelligence/TodayPage.tsx:66-81`

**Problem:** The current render logic at line 69 is:
```tsx
{!hasSubscriptions || hasError ? (
  renderEmptyState()
) : (
  <>
    <TodayArticleList ref={handleRefReady} />
    <View ... />
```

When `hasSubscriptions=true` and `hasError=false`, it always renders the article list — even if there are 0 articles. The `renderEmptyState()` function at line 56-64 handles `no_new_articles` but it's unreachable because the ternary never enters the empty-state branch in that case.

**Fix:** Use `useArticle`'s `isEmpty` return value to also trigger empty state rendering.

The full `renderEmptyState` function (lines 56-64) already handles all three cases correctly — it just wasn't reachable for `no_new_articles`. No changes needed there.
